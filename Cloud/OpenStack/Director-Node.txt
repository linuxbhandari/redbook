Director Node (Undercloud)
	- The Director node is used for deployment and management of the OpenStack environment.
	- Simple Defination : Director node deploys and manages the OpenStack overcloud.
	It is the server used to:
		Deploy OpenStack
		Manage OpenStack lifecycle
		Provision bare metal nodes

Common Services on Director Node
	Service				Purpose
	OpenStack Heat		Heat performs orchestration using templates.
	OpenStack Ironic	Ironic provides bare metal provisioning.
	Ansible				Configuration
	Podman/Docker		Containerized services
	OpenStack Nova 		Because the undercloud itself is an OpenStack environment, and Nova is required for deployment/orchestration operations.


Common Director Commands
	#openstack undercloud install
	#openstack overcloud deploy
	#openstack baremetal node list

Q How do you deploy overcloud?
	# openstack overcloud deploy

Q) How are nodes discovered?
	# Using Ironic introspection and provisioning.

Q) How do you check overcloud status?
	#openstack server list
	#openstack baremetal node list
Q) How do you view deployment logs?
	/var/log/containers/
	/var/log/heat/
	/var/log/ironic/

Q) What is introspection?
	Process where Ironic collects hardware details of bare metal nodes.
Q) What containers are used in RHOSP?
	Podman
Q) How do you scale compute nodes?
	Add node definitions and redeploy overcloud.

	Import node → introspect → assign profile → increase ComputeCount → redeploy overcloud.

Common Troubleshooting
	Problem	Cause
	Node stuck manageable	Introspection issue
	Deployment failed		Network/DNS/IPMI issue
	Compute not visible		nova-compute failed	
	VM not scheduling		Placement/resource issue

Q) Explain complete RHOSP deployment flow
Director node (undercloud)
    ↓
Ironic provisions bare metal
    ↓
Heat orchestrates deployment
    ↓
Controller/compute nodes deployed
    ↓
Overcloud becomes operational
