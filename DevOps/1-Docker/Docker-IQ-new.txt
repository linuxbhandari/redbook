Q) What is Docker?
	Docker is a containerization platform used to package applications and their dependencies into lightweight containers.

Q) What is a Container?
	A container is an isolated runtime environment that contains:
		- application code
		- libraries
		- dependencies

	Containers share the host OS kernel.

Q) Difference between VM and Container
	VM						Container
	Full OS					Shares host kernel
	Heavy					Lightweight
	Slow startup			Fast startup
	More resource usage		Less resource usage

Q) What is Docker Image?
	Docker image is a read-only template used to create containers.
	Example:
		- nginx image
		- ubuntu image
		
Q) What is Dockerfile?
	A Dockerfile is a text file containing instructions to build a Docker image.

Q) What is Docker Hub?
	Docker Hub is a public registry used to store and share Docker images.

Q) What happens when you run docker run?
	Docker:
		Pulls image (if missing)
		Creates container
		Allocates filesystem/network
		Starts container process

Q) Explain this Dockerfile

FROM nginx:latest
COPY index.html /usr/share/nginx/html/index.html
EXPOSE 80
RUN apt update & apt install vim & touch /tmp/manish
CMD ["nginx", "-g", "daemon off;"]



	Explanation:
	FROM → base image
	COPY → copy file
	EXPOSE → open port
	CMD → start nginx

Q) Container Layer Architecture
	
	1. Read-Only Layers (Lower Layers)
		- These are the layers created from the Docker image.
		- They include the OS, dependencies, and application code.
			They are immutable (cannot be changed)
			Shared across multiple containers
			Created from Dockerfile instructions (FROM, RUN, COPY)

	2. Writable Layer (Upper Layer)
		When a container is started, Docker adds a thin writable layer on top of the image layers.
			Stores runtime changes like:
				modified files
				logs
				temporary data
				newly created files
		Changes do NOT affect the original image

	In short
		A Docker container consists of multiple read-only image layers (lower layers) and a single writable layer (upper layer) where all runtime changes are stored.

	Explain COW
	Copy-on-Write means when a file from a read-only image layer is modified inside a container, Docker does not change the original file. Instead, it copies the file into the writable layer and applies changes there.

	Correct understanding
	When you write data:

	1. Inside container filesystem (no volume)
		Uses Copy-on-Write (CoW)
		Data goes to writable layer
		Container size increases

	2. Inside volume
		CoW is NOT used ❌
		Writable layer is NOT used ❌
		Data goes directly to external storage (volume)

		Correct Mental Model
			Without volume:
			App → Writable Layer (CoW applies)

			With volume:
			App → Volume Storage (bypasses writable layer)

		Docker volumes are important because they bypass the container writable layer and store data externally. This prevents container size growth and ensures data persistence even if the container is deleted.
		
Q) How to create and use volume?
	docker run -v myvol:/app/data myimage
