Q) What is ConfigMap in Kubernetes?
	ConfigMap is used to store non-sensitive configuration data in key-value format.

Q) What is Secret in Kubernetes?
	Secret is used to store sensitive data like passwords, tokens, and keys in encoded format.

Q) Difference between ConfigMap and Secret?
	ConfigMap					Secret
	Stores non-sensitive data	Stores sensitive data
	Plain text					Base64 encoded
	Example: app config			Example: DB password

Q) Why do we use ConfigMap?
	- To separate configuration from application code
	- To avoid rebuilding images for config changes

Q) Why do we use Secret?
	- To securely store credentials
	- To avoid hardcoding passwords in code

Q) How can ConfigMap be used in a Pod?
	- Environment variables
	- Volume mount

Q) How can Secret be used in a Pod?
	- Environment variables
	- Volume mount

Q) Is Secret fully secure in Kubernetes?
	- No, Secrets are only base64 encoded, not encrypted by default.

Q) How does Kubernetes store Secrets?
	Stored in etcd (can be encrypted at rest if configured).

Q) What happens if ConfigMap is updated?
	- Pods do NOT automatically restart (unless mounted as volume with sync delay or restarted manually).

Q) Can ConfigMap be used for binary data?
	- No, use Secret or external storage.

Q) How are Secrets encoded?
	Base64 encoding (not encryption)

Q) What is immutable ConfigMap/Secret?
	- Once set, cannot be changed (helps performance and safety).


Scenario-Based Questions (VERY IMPORTANT)
=========================================
Q) How do you update ConfigMap in production?
	Update ConfigMap
	Restart Pods (if env variables used)
	Or use volume mount for dynamic updates

Q) Secret is exposed in logs — what will you do?
	- Check logging configuration
	- Remove sensitive data from logs
	- Use RBAC to restrict access

Q) Pod is not getting ConfigMap values — why?
	Possible reasons:
		- Wrong key name
		- ConfigMap not in same namespace
		- Pod not restarted after update

Q) How do you debug Secret issues?
	#kubectl describe secret <name>
	#kubectl get secret <name> -o yaml

Q) ConfigMap updated but Pod still shows old value — why?
	Because env variables are injected at startup; Pod restart is required.

Q) How to decode Secret?
	#echo cGFzc3dvcmQ= | base64 --decode

Q) Why not store secrets in Docker image?
	- Because images are shared and not secure.
