Workload Objects
================
2. Pod : A Pod is the smallest deployable unit in Kubernetes.
	A pod contains:
		One or more containers
		Shared network	
		Shared storage
		Same lifecycle
	
	Containers inside a pod communicate using localhost.
	
	- Multi-container pod : A pod can contain multiple containers.
		All containers share:
			Same IP
			Same network namespace
			Volumes/storage
	
	- Init containers : Init containers run BEFORE main application containers.
		Purpose:
			Setup tasks
			Dependency checks
			Database readiness
			Configuration download
			
	- Sidecar containers : A sidecar container runs alongside the main application container.
	
	- Pod lifecycle : A pod goes through multiple phases during its lifetime.
		Flow  : Pending → Running → Succeeded/Failed
	
	Pod states
		Pending : 
		Running : 
		Succeeded
		Failed
		Unknown
