A Deployment ensures that a specified number of identical Pods are running at all times and provides features like rolling updates, rollback, and self-healing.

	A Deployment ensures:
		Desired number of replicas are running
		Pods are recreated if they fail
		Updates happen without downtime
		Rollback is possible if something breaks


Architecture Flow
	Deployment
    	↓
	ReplicaSet
    	↓
	Pods

Important Features
1. Self-healing :
	If a Pod dies:
		👉 Deployment automatically recreates it

2. Scaling
	Increase or decrease replicas:
	# kubectl scale deployment nginx-deployment --replicas=5

3. Rolling Updates
	When you update image:
		kubectl set image deployment nginx-deployment nginx=nginx:1.26

	Kubernetes:
		updates pods gradually
		avoids downtime

4. Rollback
	If update fails:
		kubectl rollout undo deployment nginx-deployment
