Q) kubernetes 
	- Kubernetes is an open-source container orchestration platform used to automate deployment, scaling, networking, and management of containerized applications.

Q) Main components of Kubernetes architecture?
	Control Plane Components
		API Server
		Scheduler
		Controller Manager
		etcd
	
	Worker Node Components
		Kubelet : Kubelet is an agent running on each node that ensures containers are running as expected.
		Kube Proxy
		Container Runtime (Docker/containerd)

Q) What is a Pod?	
	A Pod is the smallest deployable unit in Kubernetes. It can contain one or more containers sharing:
		Network namespace
		Storage volumes
		IP address

Q) What is a Node?
	A Node is a worker machine (VM or physical server) where Pods run.

Q) What is Namespaces 
	- Namespaces logically isolate resources within a cluster.
	Example:
		dev,test,production

Q) What is a Deployment?
	A Deployment manages stateless applications and provides:
		Rolling updates
		Rollbacks
		Desired state management
		Replica management

Q What is ReplicaSet?
	ReplicaSet ensures the specified number of Pod replicas are running.

Q)Difference between Deployment and StatefulSet?
	Feature				Deployment			StatefulSet
	Application type	Stateless			Stateful
	Pod identity		Random				Stable (fixed names)
	Storage				Shared / ephemeral	Persistent per Pod
	Scaling	Parallel	Ordered
	Pod replacement		New identity		Same identity
	Use case			APIs, web apps		Databases, clusters

Q) What is a Service in Kubernetes?
	A Service exposes Pods internally or externally.
	Types:
		ClusterIP
		NodePort
		LoadBalancer
		ExternalName

Q) What is ConfigMap?
	ConfigMap stores non-sensitive configuration data separately from application code.

Q) What is Secret?
	Secrets store sensitive information such as:
		Passwords
		Tokens
		API keys

Q) What is Ingress?
	Ingress manages external HTTP/HTTPS access to services with routing rules.

Q) What are Labels and Selectors?
	- Labels are key-value metadata attached to objects.
	- Selectors filter resources based on labels.

Q) What is a DaemonSet?
	DaemonSet ensures one Pod runs on every node.
	Examples:
		Log collectors
		Monitoring agents

Q) What is a Job and CronJob?
	Job → Runs a task once
	CronJob → Runs tasks on schedule

Q) What are Taints and Tolerations?
	They control Pod scheduling on nodes.
		Taint → Restricts nodes
		Toleration → Allows Pod to schedule

Q) Explain Kubernetes networking model.
	Every Pod gets unique IP
	Pods communicate directly
	No NAT required between Pods
	CNI plugins handle networking

	Examples:
		Calico
		Flannel
		Cilium
Q) What is CNI?
	Container Network Interface standard for networking plugins.

Q) What happens during kubectl apply?
	YAML sent to API server
	Object stored in etcd
	Scheduler assigns node
	Kubelet creates Pod
	Desired state maintained

Q) Explain rolling updates in Kubernetes.
	Rolling updates gradually replace old Pods with new Pods without downtime.
	
Q) How do you secure Kubernetes clusters?

	Secrets management
	TLS everywhere
	Image scanning

Q) Explain RBAC in Kubernetes.
	Role-Based Access Control defines permissions for users/services.
	Components:
		Role
		ClusterRole
		RoleBinding
		ClusterRoleBinding

Q) What are access modes in Kubernetes storage?
	Access Mode	Meaning
		ReadWriteOnce (RWO)	Mounted by single node
		ReadOnlyMany (ROX)	Read-only by many nodes
		ReadWriteMany (RWX)	Read-write by many nodes
Q) What is reclaim policy?
	Defines what happens after PVC deletion.
	Types:
		Retain
		Delete
		Recycle (deprecated)

Q) What are common storage issues in Kubernetes?	
	PVC pending
	Mount failures
	Permission denied
	Disk pressure
	StorageClass mismatch
	CSI driver failure
