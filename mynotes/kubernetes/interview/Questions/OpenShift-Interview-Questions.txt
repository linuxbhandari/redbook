Kubernetes Networking Explained

Kubernetes networking is the system that allows:
	* Pods to communicate with each other
	* Services to expose applications
	* External users to access applications
	* Traffic routing and load balancing
	* Secure communication inside the cluster
	
	Kubernetes networking is built around a flat network model.


Core Networking Principles
	Kubernetes defines these networking rules:
	1. Every Pod Gets a Unique IP
		- Each Pod has its own IP address.
		- Pods communicate directly using these IPs.
	
	2. Pods Can Communicate Without NAT
		- Any pod can directly reach another pod.
			# curl http://10.244.2.8:8080
		- No port mapping required.

	3. Containers Inside Same Pod Share Network
		Containers in the same pod:
		- share IP
		- share localhost
		- share network namespace

1. Pod Networking
	Pods are the smallest deployable units.
	Each pod gets:
	- Unique IP
	- Virtual network interface
	- Internal routing

   Pod-to-Pod Communication
   Same Node
	Pod A → Linux bridge → Pod B
	
	Uses:
	- veth pairs
	- bridge network
	- iptables

	Different Nodes
	Pod A → Node A → CNI overlay network → Node B → Pod B
	
	Handled by the CNI plugin.

2. CNI (Container Network Interface)
	CNI plugins implement networking.
	Popular CNIs:
	- Calico
	- Flannel
	- Cilium
	- Weave Net

	What CNI Does
	CNI plugin handles:
	- Pod IP allocation
	- Routing
	- Overlay networking
	- Cross-node communication
	- Network policies
	
	Without CNI:
	- Pods cannot communicate

3. Services
	Pod IPs are temporary.
	If a pod restarts:
	- Old IP gone
	- New IP assigned
	
	So Kubernetes uses Services.

    Service Types
	ClusterIP (Default)
	- Internal communication only.

	NodePort
	Exposes service on node port.

	LoadBalancer
	- Creates cloud load balancer.
	
	Used in:
	- AWS
	- Azure
	- GCP

	ExternalName
	- Maps service to external DNS.

4. kube-proxy
	kube-proxy runs on every node.
		Responsibilities:
		- Service routing
		- Load balancing
		- iptables/ipvs rules

	Traffic Flow Example
	
	Client → Service IP → kube-proxy → Backend Pod

5. DNS in Kubernetes
	Kubernetes includes DNS using:
	CoreDNS

6. Ingress
	Ingress exposes HTTP/HTTPS routes.
		Instead of:
				Many LoadBalancers
	
		Ingress provides:
				Single entry point

7. Network Policies
	Control traffic between pods.
		Without policies:
		
		All pods can talk to all pods
